> **For entity lookups and global rules**: See `../agents_index.yaml`
>
> **Index Updates**: If adding a new report type or folder structure, update `../agents_index.yaml` `folders` registry and update `meta.last_updated` and `changelog`.

This folder contains generated reports and status updates.

## Structure

```
reports/
├── sagemaker/           # SageMaker endpoint monitoring reports
│   └── YYYY-MM-DD_HH-MM-SS/
│       └── report.md
└── status_updates/      # Weekly team status reports
    └── project_updates_YYYY-WXX.md
```

## Report Types

| Type | Location | Template | Generated By |
|------|----------|----------|--------------|
| Weekly Status | `status_updates/` | `../templates/weekly_status_report.md` | `../scripts/github_weekly_report.py` |
| SageMaker Health | `sagemaker/` | None (auto-generated) | `../scripts/sagemaker_report.py` |

## Weekly Status Reports

- **Naming**: `project_updates_YYYY-WXX.md` (ISO week number)
- **Content**: Aggregated project status, highlights, blockers
- **Generation**: Run `python ../scripts/github_weekly_report.py`

## SageMaker Reports

- **Naming**: `YYYY-MM-DD_HH-MM-SS/report.md`
- **Purpose**: Endpoint health, inference metrics, errors
- **Generation**: Run `python ../scripts/sagemaker_report.py`

## Cross-References

When reports mention:
- **People**: Consider updating their `../communication/people/<name>/references.md`
- **Projects**: Link to `../project_management/<project>/`
- **Tickets**: Reference `../tickets/<TICKET-ID>.md`
